# Product Brief: flow-orche

## Executive Summary

flow-orche는 Claude Code Agent Teams를 시각적으로 관리하는 Tauri 기반 데스크톱 애플리케이션이다.
개인 개발자가 프로젝트별로 사전 구성된 AI 에이전트 팀을 원클릭으로 구성하고,
SM(Scrum Master) 에이전트 중심의 자동 오케스트레이션을 통해
개발-테스트-리뷰 파이프라인을 병렬로 실행할 수 있게 한다.

프로젝트를 선택하면 tmux 세션이 자동 생성되고 Claude Code가 실행되며,
프로젝트의 기술 스택과 스킬을 분석하여 최적의 팀 구성을 제안한다.
태스크 의존성 기반으로 각 단계가 자동 진행되고,
다중 계정 지원으로 대규모 작업량에 대한 유연한 리소스 전환이 가능하다.

AI 모델이 발전할수록 각 에이전트의 코드 품질과 협업 능력이 향상되어
앱의 가치도 자연스럽게 상승하는 "AI 레버리지 플랫폼"이다.

## Core Vision

### Problem Statement

Claude Code의 Agent Teams는 강력한 멀티 에이전트 오케스트레이션을 제공하지만,
현재 CLI 기반으로만 동작한다. 개인 개발자가 여러 프로젝트를 병렬로 관리할 때
매번 터미널을 열고, 프로젝트 디렉토리로 이동하고, Claude를 실행하고,
스킬을 로드하고, 팀을 구성하는 반복 작업이 발생한다.
작업량이 많아 다중 계정을 사용하는 경우 전환 과정이 번거롭고,
에이전트 간 작업 완료 감지를 타이머나 수동 확인에 의존해야 한다.

### Problem Impact

- 프로젝트 전환마다 수분의 세팅 시간 소요
- SM 에이전트의 작업 완료 포착이 불안정 (타이머/수동 방식)
- 계정 전환 시 세팅 유실 불안 및 번거로움
- 팀 구성을 매번 처음부터 지시해야 하는 비효율
- 에이전트 팀의 진행 상황을 한눈에 파악하기 어려움

### Why Existing Solutions Fall Short

- **터미널 + tmux 수동**: 프로젝트별 설정 저장 불가, 계정 관리 불가, 시각적 모니터링 불가
- **VS Code Claude Extension**: 단일 세션만 지원, Agent Teams 미지원
- **Cursor / Windsurf**: Claude Code 생태계(스킬, BMAD, Agent Teams)와 호환 불가
- **Claude Code CLI 단독**: GUI 없음, 프로젝트 관리 없음, 계정 전환이 번거로움, 프리셋 개념 없음

어떤 기존 도구도 "프로젝트별 AI 에이전트 팀 오케스트레이션"을 시각적으로 관리하는 것을 제공하지 않는다.

### Proposed Solution

Tauri 기반 데스크톱 앱으로 다음을 제공한다:

1. **프로젝트 허브**: 프로젝트 등록/관리, 프로젝트별 설정(스킬, CLAUDE.md) 저장
2. **원클릭 팀 구성**: 프로젝트 선택 시 tmux 세션 자동 생성 → Claude Code 실행 → tmux send-keys로 Agent Teams 자동 스폰
3. **자동 프리셋**: Claude가 프로젝트의 기술 스택, 스킬 파일을 분석하여 최적 팀 구성 제안 (MVP: 수동 저장, v2: 자동 추천)
4. **SM 중심 오케스트레이션**: 태스크 의존성 그래프 기반으로 개발 → 테스트 → 리뷰 자동 진행, 실패 경로(테스트 실패/리뷰 reject/에이전트 stuck) 대응 포함
5. **다중 계정 관리**: Max*20 계정 간 빠른 전환, 프로젝트 데이터 독립 보장, OAuth 인증은 시스템 브라우저 위임
6. **실시간 대시보드**: 태스크 상태, 에이전트 활동, 의존성 그래프 시각화
7. **환경 자동 감지**: 첫 실행 시 Claude Code CLI, tmux, 인증 상태를 자동 체크하고 안내

### Key Differentiators

- **Claude Code Agent Teams 네이티브 통합**: Agent Teams의 파일 기반 아키텍처(~/.claude/teams/, ~/.claude/tasks/)를 직접 활용하는 유일한 GUI 도구
- **tmux send-keys 기반 원클릭 자동화**: 프로젝트 선택 → tmux 세션 생성 → Claude Code 실행 → 팀 프롬프트 자동 전송의 완전 자동화
- **프로젝트 자동 분석 → 팀 프리셋**: 프로젝트 구조와 스킬을 읽어 최적 팀 구성을 자동 제안
- **SM 에이전트 패턴 내장**: 개발-테스트-리뷰 파이프라인의 자동 오케스트레이션 (실패 경로 포함)
- **AI 발전에 따른 자연스러운 가치 증가**: Claude 모델이 발전할수록 각 에이전트의 코드 품질과 협업 능력이 향상 → 앱의 가치도 자동 상승
- **개인 개발자 특화**: 1인 개발자의 멀티 에이전트 워크스테이션으로서 생산성 극대화에 초점

## Target Users

### Primary Users

#### Persona: "승민" — Claude Code 파워유저 솔로 개발자

**프로필:**
- 개인 개발자, 항상 개발을 하는 사람
- Claude Code Max*20 구독 계정 2개 보유
- 동시에 5~8개 프로젝트를 관리 (5~6개 상시 활성)
- BMAD Method로 계획 → PRD → 아키텍처 → 개발 파이프라인을 체계적으로 운영
- 회사 프로젝트와 개인 프로젝트를 병행

**현재 워크플로우:**
- tmux에서 SM 에이전트를 두고 역할별 에이전트에게 작업을 분배
- Git worktree를 활용하여 병렬 작업 시 파일 충돌 방지
- 코드 리뷰는 BMAD code review + 보안 스킬 + PR review 스킬을 조합한 올인원 방식
- 토큰이 소진되면 두 번째 계정으로 전환하여 작업 지속

**Pain Points:**
- 프로젝트 전환마다 터미널 세팅 반복 (디렉토리 이동, claude 실행, 스킬 로드, 팀 구성)
- 계정 전환 과정이 번거로움
- 에이전트 간 작업 완료 감지가 수동/타이머 기반
- 매번 팀 구성을 처음부터 지시해야 함
- 여러 프로젝트의 진행 상황을 한눈에 파악하기 어려움

**성공 비전:**
- BMAD로 철저히 계획한 후, flow-orche에서 원클릭으로 에이전트 팀이 구성되어
  에러 없이 의도한 대로 오케스트레이션하며 앱이 빠르게 완성되는 것
- SM이 알아서 Dev에게 스토리를 할당하고, worktree를 생성하고,
  완료되면 QA가 자동으로 리뷰하는 파이프라인

**일반적인 팀 구성 패턴:**
- 기본: SM(1) + Dev(1) + QA(1)
- 확장: SM(1) + Dev(2~3, 영향 범위가 독립적일 때) + QA(1~2)
- 각 Dev는 별도 git worktree에서 작업하여 파일 충돌 방지
- 작업 완료 후 팀 해산, worktree 정리

### Secondary Users

#### 잠재적 미래 사용자: Claude Code 하드유저

- Claude Code를 일상적으로 사용하는 개인 개발자
- Max 구독을 보유하고 있으며, 멀티 에이전트 워크플로우에 관심이 있는 사용자
- Agent Teams 기능을 활용하고 싶지만 CLI 진입장벽이 높은 사용자
- 현재는 타겟팅하지 않으나, 제품이 안정되면 자연스럽게 확장 가능한 세그먼트

### User Journey

#### 1. Discovery (발견)
- 본인이 직접 필요를 느끼고 만드는 도구이므로 발견 단계 없음
- 미래 사용자의 경우: Claude Code 커뮤니티, GitHub에서 발견

#### 2. Onboarding (첫 경험)
1. flow-orche 설치 및 실행
2. 환경 자동 감지 (Claude Code CLI, tmux, 인증 상태 확인)
3. Max*20 계정 2개 등록
4. 기존 프로젝트 디렉토리를 프로젝트 카드로 등록
5. 프로젝트별 팀 프리셋 구성 (SM + Dev + QA)

#### 3. Core Usage (일상 사용)
1. flow-orche 실행 → 프로젝트 카드 목록 확인
2. 작업할 프로젝트 선택 → [시작] 클릭
3. tmux 세션 자동 생성 → Claude Code 실행 → 팀 자동 구성
4. SM이 BMAD 계획에 따라 스토리 할당 → Dev가 worktree에서 작업
5. 작업 완료 시 QA 자동 트리거 (보안 + 코드리뷰 + PR 리뷰)
6. 토큰 소진 시 계정 전환 (프로젝트/팀 유지)
7. 작업 완료 후 팀 해산, worktree 정리

#### 4. Success Moment ("아, 만들길 잘했다")
- BMAD로 계획한 PRD의 스토리들이 에이전트 팀에 의해 자동으로 구현되고,
  QA 리뷰까지 통과하여 머지되는 것을 대시보드에서 지켜보는 순간
- 5개 프로젝트를 전환하며 각각의 진행 상황을 한눈에 파악하는 순간

#### 5. Long-term (장기 사용)
- 프로젝트별 최적 팀 프리셋이 축적됨
- 프로젝트 유형별 패턴 학습 (프론트엔드 프로젝트는 이런 팀, 백엔드는 저런 팀)
- AI 모델 업그레이드에 따라 에이전트 품질 자연 향상

## MVP Scope

### Core Features

#### MVP 필수 기능 (v1.0)

**F1. 프로젝트 허브 (Project Hub)**
- 프로젝트 등록: 로컬 디렉토리 경로 지정으로 프로젝트 카드 생성
- 프로젝트 목록: 카드 형태로 등록된 프로젝트 표시 (이름, 경로, 상태, 프리셋)
- 프로젝트별 설정 저장: 팀 프리셋, 사용할 계정, 기본 프롬프트 등
- 프로젝트 상태 표시: 활성(tmux 세션 실행 중) / 비활성

**F2. 원클릭 팀 실행 (One-Click Launch)**
- 프로젝트 카드에서 [시작] 클릭 시:
  1. tmux 세션 자동 생성 (`tmux new-session -d -s {project-name}`)
  2. 프로젝트 디렉토리로 이동
  3. 환경변수 설정 (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)
  4. Claude Code 실행 (`claude --teammate-mode tmux`)
  5. Claude 준비 상태 감지 (프롬프트 `>` 폴링)
  6. 저장된 팀 프리셋 프롬프트 자동 전송 (`tmux send-keys`)
- 프로젝트 카드에서 [중지] 클릭 시 tmux 세션 정리

**F3. Agent Teams 자동 구성 (Team Preset)**
- 프로젝트별 팀 프리셋 수동 저장/편집
  - 팀 구성: SM(1) + Dev(N) + QA(N) 형태로 정의
  - 각 역할별 프롬프트 커스터마이즈 가능
  - 기본 프리셋 제공 (SM + Dev + QA)
- 시작 시 프리셋 프롬프트를 Claude Code에 자동 전달

**F4. 프로젝트별 알림 (Project-Aware Notifications)**
- 각 프로젝트의 Claude hooks 이벤트를 앱 내에서 수신
  - 작업 완료 (Stop 이벤트)
  - 사용자 입력 대기
  - 에러 발생
- 앱 내 알림 패널에서 프로젝트별로 구분하여 표시
- 시스템 네이티브 알림 (macOS Notification Center) 연동
- 현재 Discord 전역 알림의 한계 해결

**F5. 다중 계정 관리 (Account Switching)**
- Max*20 계정 프로필 2개 이상 등록
- 프로필별 이름 부여 (예: "계정A", "계정B")
- 원클릭 계정 전환 (claude auth 기반)
- 전환 시 프로젝트 데이터/프리셋 완전 독립 보장
- 실행 중인 팀이 있을 때 전환 경고

### Out of Scope for MVP

| **오픈소스 배포 준비** (문서, CI/CD 등) | 개인 도구 안정화 후 검토 | 12개월+ |

### MVP Success Criteria

MVP가 성공적이라고 판단하는 기준:

1. **일상 대체**: 기존 "터미널 + tmux 수동 세팅" 워크플로우를 flow-orche가 완전히 대체
2. **5개 프로젝트 관리**: 5개 이상의 활성 프로젝트를 flow-orche에서 등록하고 실행/관리
3. **원클릭 동작**: 프로젝트 카드 클릭 → 팀 구성 완료까지 추가 수동 개입 없이 90% 이상 성공
4. **알림 분리**: 각 프로젝트의 이벤트가 명확히 구분되어 알림
5. **계정 전환 무결성**: 전환 후 프로젝트 데이터/프리셋 100% 유지
